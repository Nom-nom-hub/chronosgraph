# Security Policy

## Reporting a Vulnerability

We take the security of ChronosGraph seriously. If you discover a security vulnerability within ChronosGraph, please report it to us as soon as possible.

**Please DO NOT open a public GitHub issue.**

Instead, please send an email to [INSERT SECURITY CONTACT EMAIL HERE] with a detailed description of the vulnerability, including:

*   A clear and concise description of the vulnerability.
*   Steps to reproduce the vulnerability.
*   The potential impact of the vulnerability.
*   Any suggested mitigations or fixes.

We will acknowledge your report within 48 hours and provide a more detailed response within 5 business days. We will keep you informed of our progress in addressing the vulnerability.

## Disclosure Policy

We kindly request that you do not disclose any vulnerabilities publicly until we have had a chance to address them. We believe in responsible disclosure and will work with you to ensure a timely and coordinated release of information.

## Supported Versions

The following versions of ChronosGraph are currently supported with security updates:

| Version | Supported          |
| :------ | :----------------- |
| Latest  | Yes                |
| Older   | No (Upgrade Recommended) |

We appreciate your efforts to responsibly disclose your findings and help us keep ChronosGraph secure.
